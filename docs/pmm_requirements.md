# Требования к внешней библиотеке pmm [CR-001](../requirements/constraints/CR-001.json)

Данный документ определяет требования к библиотеке [PersistMemoryManager (pmm)](https://github.com/netkeep80/PersistMemoryManager), необходимые для реализации pjson. Файл предназначен для передачи в проект pmm и представляет собой реализацию pmm с точки зрения pjson.

## Управление персистентной памятью [BR-001](../requirements/business/BR-001.json)

pmm должна предоставлять управление персистентным адресным пространством (ПАП), в котором pjson хранит все JSON-данные в бинарном образе памяти.

### Выделение и освобождение памяти [FR-001](../requirements/functional/FR-001.json)

pmm должна обеспечивать выделение и освобождение блоков памяти для хранения JSON-узлов через лес AVL-деревьев. pjson не должен дублировать управление памятью — все операции делегируются pmm.

### Адресация на основе смещений [NFR-002](../requirements/nonfunctional/NFR-002.json)

Все межобъектные ссылки должны использовать индексы гранул (смещения от базы области памяти), а не сырые указатели. Это обеспечивает адресную независимость образа памяти — возможность сохранения на диск и загрузки по другому базовому адресу.

## Лес AVL-деревьев [BR-002](../requirements/business/BR-002.json)

pmm должна реализовывать единую архитектуру леса AVL-деревьев для управления памятью и хранимыми объектами.

### Выделенные AVL-деревья для типов узлов [FR-009](../requirements/functional/FR-009.json)

pmm должна поддерживать создание выделенных AVL-деревьев с фиксированными идентификаторами типов для каждого типа узла pjson:

| node_type | Тип узла pjson | Контейнер pmm        |
|-----------|----------------|----------------------|
| 1         | $ref           | pstringview          |
| 2         | $base64        | parray\<uint8_t\>    |
| 3         | pstring        | pstring              |
| 4         | parray         | parray\<pjson_node\> |
| 5         | pjson          | pjson_node           |

### Итераторы по типизированным AVL-деревьям [FR-010](../requirements/functional/FR-010.json)

pmm должна предоставлять итераторы для обхода типизированных AVL-деревьев, позволяющие перебрать все узлы определённого типа за O(n).

## Персистентные контейнеры [SR-002](../requirements/stakeholder/SR-002.json)

pmm должна предоставлять набор персистентных контейнеров, которые pjson использует как тонкая обёртка.

### pmap — отсортированная карта на AVL-дереве [FR-001](../requirements/functional/FR-001.json)

pmm должна предоставлять `pmap<K, V>` — персистентную карту на основе AVL-дерева с поиском по ключу за O(log n). В контексте pjson ключ K предположительно всегда равен `pstringview`.

### parray — непрерывный массив [FR-001](../requirements/functional/FR-001.json)

pmm должна предоставлять `parray<T>` — персистентный непрерывный массив с доступом по индексу за O(1). Используется для JSON-массивов (`parray<pjson_node>`) и бинарных данных (`parray<uint8_t>`).

### pstring — изменяемая строка [FR-001](../requirements/functional/FR-001.json)

pmm должна предоставлять `pstring` — персистентную изменяемую строку динамического размера для хранения строковых значений JSON.

### pstringview — интернированная строка [FR-007](../requirements/functional/FR-007.json)

pmm должна предоставлять `pstringview` — персистентное интернированное строковое представление с O(1) сравнением. Используется для $ref ссылок и строковых ключей объектов.

## Бэкенды хранения [NFR-001](../requirements/nonfunctional/NFR-001.json)

pmm должна предоставлять различные бэкенды хранения для работы с персистентным адресным пространством.

### HeapStorage [NFR-001](../requirements/nonfunctional/NFR-001.json)

Хранение в оперативной памяти (куча) для тестирования и временных данных.

### MMapStorage [NFR-001](../requirements/nonfunctional/NFR-001.json)

Хранение через отображение файлов в память (mmap) для полноценной персистентности.

### StaticStorage [NFR-001](../requirements/nonfunctional/NFR-001.json)

Хранение в статической памяти для встраиваемых систем и предзаполненных данных.

## Индексация и поиск [FR-011](../requirements/functional/FR-011.json)

pmm должна обеспечивать базовую инфраструктуру для реализации индексации и полнотекстового поиска в pjson_db.

### Поиск по типизированным деревьям [FR-011](../requirements/functional/FR-011.json)

Раздельные AVL-деревья для `pstringview` и `pstring` должны позволять реализовать поиск по ключам и значениям независимо.

## Восстановление данных [FR-013](../requirements/functional/FR-013.json)

pmm должна обеспечивать возможность независимого обхода каждого типизированного AVL-дерева для восстановления неповреждённых узлов при повреждении ПАП.

## Потокобезопасность [NFR-001](../requirements/nonfunctional/NFR-001.json)

pmm должна предоставлять настраиваемые политики блокировок для поддержки потокобезопасности в pjson.
